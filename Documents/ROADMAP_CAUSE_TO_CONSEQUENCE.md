# ROADMAP: От причины к следствию

> Ключевой инсайт: модель предсказывает ПРИЧИНУ (дефицит железа),
> но продаём мы СЛЕДСТВИЕ (депрессия, презентеизм, больничные).
> Порядок важен: сначала чистое, потом ценное.

---

## Цепочка ценности

```
ПРИЧИНА (биомаркер)              СЛЕДСТВИЕ (что волнует покупателя)
       |                                    |
  Дефицит железа                  Усталость, апатия
  (Body Iron < 0)                 Депрессия (PHQ-9 выше)
       |                          Низкая самооценка здоровья
  Доказывается из ОАК             Плохая диета (замкнутый круг)
  (MCV, RDW, MCH...)                   |
       |                          Презентеизм
       |                          Больничные
       |                          Текучесть кадров
       |                                |
  ЭТО МЫ ПРОДАЁМ                 ЭТО МЫ ДОКАЗЫВАЕМ
  СЕЙЧАС                          ПОТОМ
```

---

## Этап 1: "Причина" (MVP, сейчас)

**Модель**: ОАК -> P(дефицит железа)

**Фичи**: CBC + BIOPRO + DEMO + BMX + BPX
**Таргет**: Body Iron < 0 (формула Cook, ферритин + sTfR)

**DPQ / HSQ / DBQ**: НЕ используем ни как фичи, ни как таргет

**Продаём**: "Мы находим скрытый дефицит железа из рутинного ОАК"

---

## Этап 2: "Доказательство следствия" (через 3-6 мес)

**Не новая модель**, а аналитическая надстройка.

В NHANES у каждого человека (по SEQN) есть:
- Лаб (CBC + ферритин + sTfR) -> мы уже знаем Body Iron
- Депрессия (DPQ_L: PHQ-9, шкала 0-27)
- Самооценка здоровья (HSQ_L: 1=отлично ... 5=плохо)
- Диета (DBQ_L: качество питания)
- Занятость (OCQ_L: работает ли, сколько часов)
- Физ. активность (PAQ_L)

### Что делаем

Группируем по нашему предсказанию (или по Body Iron напрямую):

```python
group_risk = merged[merged['Y_IRON_DEFICIENCY'] == 1]
group_ok   = merged[merged['Y_IRON_DEFICIENCY'] == 0]

# PHQ-9 (депрессия): 0-4 норма, 5-9 лёгкая, 10-14 умеренная, 15+ тяжёлая
# HSQ (самооценка): 1=отлично, 2=очень хорошо, 3=хорошо, 4=удовл, 5=плохо
# OCQ180 (часы работы в неделю)
```

### Что получаем

Слайд для HR-директора / инвестора:

```
"Сотрудники с выявленным скрытым дефицитом железа:
 - На X баллов выше по шкале депрессии (PHQ-9)
 - В Y раз чаще оценивают здоровье как 'плохое'
 - Работают на Z часов в неделю меньше

 Стоимость бездействия: ... руб/сотрудника/год"
```

### Зачем нужна именно NHANES

Потому что там всё в одном человеке: и кровь, и PHQ-9, и занятость.
Мы можем показать корреляцию на РЕАЛЬНЫХ данных, а не на "по литературе".

## Этап 3: "Расширенная модель" (6-12 мес, когда есть свои данные)

DPQ / HSQ / DBQ становятся ФИЧАМИ в новой модели:

```
Модель 1 (MVP):     ОАК -> дефицит железа          (медицинская)
Модель 2 (новая):   ОАК + PHQ-9 + HSQ -> риск      (продуктивность)
                    презентеизма / больничных
```

Модель 2 — другой продукт. Не "найди болезнь", а "предскажи, кто уйдёт
на больничный". За это работодатели платят совсем другие деньги.

### Источник данных для Модели 2

Не NHANES, а **собственные данные из пилотов**:
- Результаты 29н (ОАК + анкета)
- PHQ-9 (можно попросить заполнить)
- Данные о больничных от работодателя
- Follow-up: подтвердился ли дефицит, начал ли лечение

Собственные данные >> NHANES для этой задачи.

---

## Почему "потом", а не "сейчас"

| Причина | Объяснение |
|---|---|
| **Чистота данных** | PHQ-9 и HSQ — самоотчёт. Люди врут. ОАК — объективные цифры. Начинать с шумных данных рискованно для репутации модели |
| **Регуляторика** | "Предсказываем депрессию из анализа крови" = скандал. "Находим дефицит железа, который по литературе связан с депрессией" = наука |
| **Каузальность** | ЖДА -> депрессия (железо нужно для синтеза дофамина/серотонина) — есть доказательная база. Но замкнутый круг: депрессия -> плохая диета -> ЖДА. Для MVP нужно одно направление |
| **Свои данные лучше** | Для модели продуктивности лучше собрать реальные данные о больничных из пилота, чем использовать NHANES (другая страна, другой контекст) |

## Данные NHANES для будущего анализа (не удалять!)

| Файл | Переменные | Для чего |
|---|---|---|
| **DPQ_L.xpt** | DPQ010-DPQ090 (9 вопросов PHQ-9) | Корреляция ЖДА с депрессией |
| **HSQ_L.xpt** | HSD010 (самооценка здоровья) | Корреляция ЖДА с субъективным состоянием |
| **DBQ_L.xpt** | Качество диеты, частота приёмов пищи | Замкнутый круг: диета <-> ЖДА |
| **OCQ_L.xpt** | OCD150, OCQ180 (часы работы) | Связь ЖДА с трудоспособностью |
| **PAQ_L.xpt** | PAD680 (сидячее время) | Физ. активность при ЖДА |

Эти файлы УЖЕ скачаны. Когда придёт время — мержим по SEQN и считаем.

---

## Литература для связи ЖДА -> депрессия / продуктивность

- Железо необходимо для синтеза дофамина (тирозин-гидроксилаза)
  и серотонина (триптофан-гидроксилаза) — нейромедиаторы настроения
- ЛДЖ (без анемии!) ассоциирован с:
  - Повышенной утомляемостью
  - Снижением когнитивных функций
  - Депрессивной симптоматикой
  - Снижением физической работоспособности
- Мета-анализы: железозаместительная терапия улучшает PHQ-9 на 2-4 балла
  даже при латентном дефиците (без анемии)

---

## Одной фразой

Сейчас: ОАК -> ЖДА (причина, чистые данные, доказуемо).
Потом: ЖДА -> депрессия / продуктивность (следствие, для продажи работодателю).
Ещё позже: замкнутый цикл с собственными данными.
DPQ/HSQ/DBQ не удаляем — просто не суём в модель на первом шаге,
а используем как доказательную базу для ROI.
