# БИЗНЕС-МОДЕЛИ: Гипотезы, анализ, статус

## Реальные выходы (не фантазийные)

```
ЕСТЬ:    Выходы на КДЛ и УКБ (реальные контакты)
ЕСТЬ:    B2C запускаем сами (лендинг, 0 зависимостей)
МОЖНО:   Попробовать Инвитро (внутренняя ИС, если подойдёт)
НЕТ:     Выходов на МИС-вендоров (пока)
ПОТОМ:   B2B к работодателям (когда есть кейс из КДЛ/B2C)
ДАЛЬШЕ:  МИС-интеграция (когда масштаб 1000+ точек, нужна ERM)
```

## Ключевое решение: одна модель, три обёртки

```
          NHANES (все фичи)
                |
        CatBoost (одна модель)
                |
    +-----------+-----------+
    |           |           |
  КДЛ/УКБ     B2C        B2B/29н
  (лаб only)  (лаб+демо) (лаб+демо+физ)
  NaN для     NaN для    Полный набор
  физ.данных  физ.данных
    |           |           |
  AUC ~0.85   AUC ~0.85   AUC ~0.90
```

Треним на ВСЕХ фичах (включая ИМТ, АД). CatBoost нативно обрабатывает NaN.
Если канал не передаёт ИМТ/АД — модель работает, просто чуть менее точно.

## Лестница зрелости

```
(1) B2C + КДЛ "умный добор"         -> proof of concept, первая выручка, метрики
(2) Масштаб КДЛ + B2B работодатели  -> деньги, масштаб
(3) Инвитро / самостоятельный рост   -> выбор пути масштабирования
(4) МИС / страховые                 -> ERM, большие чеки, длинный цикл
```

## Гипотеза A: "Биохим-аудит продуктивности" (B2B для корпоратов/ДМС)

**Статус: ЭТАП 2 (после B2C + КДЛ)**

### Что
Дешёвый базовый скрининг (уже есть в корпоративных чекапах/29н) + ML-триаж ->
точечные доборы только группе риска (железо/метаболика/иногда щитовидка).

### Кому
- Работодатели (IT, банки, промка, ритейл)
- Администраторы ДМС, wellbeing-провайдеры
- Частные клиники на потоке 29н

### Ценность
- Дешевле "полной панели всем"
- Меньше больничных/презентеизма
- Измеримый ROI в человеко-часах

### БМ (монетизация)
- **Компонент 1**: Фикс PEPY (per employee per year) за платформу и триаж
- **Компонент 2**: Лабораторный хвост оплачивается не нами (сотрудник/работодатель/ДМС)
- **Компонент 3** (опционально): Performance fee (% от инкрементальной маржи или экономии)

### Западные аналоги
- Quest Diagnostics Workforce Screening (biometric screenings, PEPY)
- Labcorp Employer Services (on-site/off-site/at-home)
- Premise Health (скрининги + клиники на территории работодателя)
- Personify Health / Virgin Pulse (wellbeing + стимулы)

### Математика для HR
```
Зарплата сотрудника: 100 000 руб/мес
Потеря эффективности из-за ЛДЖ (15%): -15 000 руб/мес убытка
Цена решения: 300 руб (единоразовый добор ферритина группе риска)
ROI: ~5000%
```

### Ключевой инсайт
ОМС/29н -> источник данных и массового базиса.
ДМС -> источник денег и быстрых контрактов.

## Гипотеза B: "Интеграция / White-label"

**Статус: ЭТАП МАСШТАБИРОВАНИЯ (не старт)**

### Что
Встраиваемый AI-модуль: подсказки, триаж доборов, pre-analytical валидация.

### Три варианта (по приоритету)

**Вариант 1 — Самостоятельность (предпочтительный)**
- Заходим в КДЛ напрямую через B2C + pitch
- Свой бренд, свой IP, максимальная маржа
- Лицензируем IP при необходимости, но не отдаём
- Подходит до ~50-100 точек

**Вариант 2 — Инвитро / крупная КДЛ**
- Если у Инвитро внутренняя ИС подходит для нашего решения — интегрируемся
- Плюсы: готовая инфра, огромный поток, быстрый масштаб
- Минусы: зависимость, они "продвиженцы решения" (решают за нас)
- Ключевое: не теряем IP, работаем на их данных но со своей моделью

**Вариант 3 — МИС-модуль (масштаб 1000+)**
- Когда 1000 КДЛ / 1000 работодателей — нужна ERM-система
- Тогда White-label для МИС (БАРС, qMS, Medods, 1С-Медицина) становится логичным
- Лицензия per-clinic / per-user + revenue-share

### Кому (в зависимости от варианта)
- Вариант 1: КДЛ/УКБ напрямую
- Вариант 2: Инвитро / Хеликс / крупная сеть
- Вариант 3: МИС-вендоры

### Ценность (для любого варианта)
- Партнёр получает дифференциацию ("ИИ-подсказки")
- Клиники получают рост ARPU + этичный апсейл
- Мы получаем доступ к рынку через масштабного партнёра

### БМ
- IP-лицензия (максимум что отдаём)
- Revenue-share с доборов
- API-доступ

### Когда
- Сейчас: фокус на B2C + КДЛ напрямую
- Когда proof of concept + метрики → переговоры с Инвитро
- Когда 100+ точек → думаем про МИС

## Гипотеза C: "Независимый аудитор назначений для страховых"

**Статус: СТРАТЕГИЧЕСКИ СИЛЬНО, НО ДЛИННЫЙ ЦИКЛ**

### Что
API-слой между клиникой и страховой: проверка обоснованности назначений и счетов.

### Кому
- Страховые, TPA, корпоративные администраторы ДМС
- Цифровые админы (Лучи/BestDoctor и подобные)

### Ценность
- Снижение перерасхода
- Меньше конфликтов
- Прозрачность клинических решений

### БМ
- SaaS-подписка для страховой
- % от предотвращённых выплат

### Почему КДЛ это не сделает
Они не могут сами себя штрафовать. Конфликт интересов.

### Российские аналоги (в смежном пространстве)
- Mains Lab — анализ счетов, выявление завышений в ДМС

## Гипотеза D (v3): "Умный добор" — инкрементальная выручка для КДЛ

**Статус: ОСНОВНАЯ РАБОЧАЯ ГИПОТЕЗА**

### Ключевой разворот

```
v1 (мёртвая):  "Мы поможем КДЛ делать МЕНЬШЕ тестов"
               КДЛ: "зачем мне меньше чеков?"

v2 (слабая):   "КДЛ — партнёр/канал, а не покупатель"
               Работает, но КДЛ непонятно зачем

v3 (РАБОЧАЯ):  "Мы поможем КДЛ продать БОЛЬШЕ тестов НУЖНЫМ людям"
               КДЛ: "рассказывай"
```

### Как работает

```
[Пациент сдаёт ОАК в КДЛ (495 руб)]
        |
        v
[Результат готов]
        |
        v
[Наша модель: P(дефицит железа) = ?]
        |
  +-----+-----+
  |             |
  v             v
P < 0.5        P >= 0.5
(~75%)         (~25%)
  |             |
Ничего         SMS / email пациенту:
               "Обнаружены изменения в ОАК.
                Рекомендуем ферритин (425 руб, скидка 15%).
                Это не диагноз."
        |
        v
  +-----+-----+
  |             |
  v             v
Согласился     Не согласился
(30-40%)       (60-70%)
  |
  v
[Сдал ферритин]
  |
  v
PPV 60-70%: дефицит подтверждён
```

### Почему ВСЕ стимулы совпадают

| Участник | Что получает | Стимул |
|---|---|---|
| КДЛ | Доп. выручка с текущего потока (без доп. затрат на привлечение) | +деньги |
| Пациент | Найденная проблема + скидка (а не "панель на всякий случай" за 7k) | +здоровье, -деньги |
| Мы | % от инкрементальной маржи КДЛ | +деньги |
| Врач | Не трогаем вообще | Нет конфликта |

### Почему нет проблемы с ответственностью

```
Мы ничего НЕ отменяем.
Мы ничего НЕ блокируем.
Мы ДОБАВЛЯЕМ рекомендацию пациенту.

FN невозможен:
  - Мы рекомендовали СДЕЛАТЬ тест, а не НЕ делать
  - Если пациент отказался — его выбор
  - Если дефицит не подтвердился — "рекомендация обоснована паттерном ОАК"
```

### Экономика (на реальных ценах, Хеликс)

```
ОАК = 495 руб (уже оплачен пациентом, мы не трогаем)
Ферритин = 500 руб (добор)
Скидка пациенту: 15% -> 425 руб

Себестоимость добора для КДЛ: ~100-150 руб
Маржа КДЛ с добора: ~275-325 руб
Наша доля: 20-30% от маржи = 55-100 руб / добор

НА 1 ТОЧКУ КДЛ (200 ОАК/день):
  Жёлтая зона: 50 чел/день (25%)
  Конверсия: 17 чел/день (30-35%)
  
  КДЛ доп. выручка: 17 × 425 = 7 225 руб/день = ~217 000 руб/мес
  КДЛ доп. маржа:   17 × 300 = 5 100 руб/день = ~153 000 руб/мес
  Нам:               17 × 75  = 1 275 руб/день = ~38 000 руб/мес

  PPV: 10-12 из 17 подтверждают дефицит (60-70%)
  = 10-12 реально довольных пациентов/день

МАСШТАБ:
  10 точек:  380 000 руб/мес нам
  50 точек:  1.9 млн руб/мес
  100 точек: 3.8 млн руб/мес (~45 млн/год)
```

### Что нужно от КДЛ (минимум для старта)

```
1. Доступ к результатам ОАК (с согласием пациента на обработку)
2. Контактные данные пациента (SMS/email)
3. Витрина доборов + цена со скидкой
4. Механизм записи на добор (ссылка / запись)

НЕ НУЖНО:
  - Интеграция в ЛИС
  - Изменение workflow врача
  - Согласование с назначающим врачом
  - Регистрация как МИ / СППВР
```

### Честная проблема: согласие пациента на контакт

Главный барьер — получить право писать пациенту после сдачи.
Решения:
- Согласие при сдаче ("вы согласны получать рекомендации по результатам?")
- Через ЛК КДЛ (если есть)
- Через SMS от имени КДЛ (не от нашего имени)
- Opt-in при регистрации в КДЛ

## Гипотеза D+: Pre-analytical Optimization (custdev)

**Статус: ИССЛЕДУЕМ. Не core, но потенциально ценный бонус-модуль.**

### Что
Валидация интерпретируемости результатов и оптимизация назначений:
- Ферритин при воспалении → рекомендовать sTfR/CRP
- Дублирующие тесты (железо сыворотки при наличии ферритина)
- Бессмысленные повторы (<14 дней при норме)
- Замена на более информативный тест

### Почему не core
- КДЛ зарабатывает на каждом тесте — "делать меньше" не продаваемо
- КДЛ не хочет конфликтовать с врачом
- Неизвестно, какой % оптимизации реально есть на рынке РФ

### Почему не убираем
- Может быть бонусной фичей для КДЛ (quality layer)
- Может быть core для страховых (Гипотеза C — "аудитор")
- Может быть фичей B2C ("ваш ферритин мог быть ложным из-за воспаления")
- Нужен custdev: поговорить с 5-10 КДЛ, понять реальную боль

### Что нужно для custdev
```
1. Интервью с 5-10 КДЛ:
   - "Как часто к вам приходят с бессмысленными назначениями?"
   - "Сколько реально пересдач в месяц и почему?"
   - "Что бы вы хотели, чтобы система делала ДО взятия крови?"

2. Данные (если дадут):
   - % результатов, которые "не интерпретируемы"
   - % повторных сдач <14 дней
   - Топ-5 самых частых "глупых" назначений

3. Результат custdev:
   - Либо: "есть боль, % оптимизации X — добавляем как модуль"
   - Либо: "боли нет, КДЛ не жалуется — оставляем в бэклоге"
```

## Гипотеза E: "Корпоративный Паспорт Здоровья" (B2B2C)

**Статус: ЖИВОЙ ВАРИАНТ, канал через КДЛ/УКБ или B2C-платформу**

### Сценарий
1. Иван (сварщик) проходит обязательный профосмотр. Сдаёт кровь по 29н. За это платит Завод.
2. Клиника ставит штамп "Годен".
3. AI анализирует расширенный ОАК Ивана:
   - Hb норма, но MCV низкий, RDW высокий -> скрытый дефицит железа
   - Глюкоза верхняя граница + возраст + вес -> риск преддиабета
4. Ивану приходит СМС: персональный риск-профиль + одно точечное дообследование

### Формулировки (регуляторно безопасные)
- НЕ: "у вас риск анемии"
- ДА: "есть изменения в показателях эритроцитов; рекомендуем уточняющий анализ"
- Обязательно: "это не диагноз", "рекомендуем обсудить с врачом"

### Кто платит
- Сотрудник (B2C-докупка)
- Работодатель (wellbeing budget)
- Клиника (получает % с апсейла)

## Гипотеза F: B2C "Загрузи анализы — получи гайдлайн"

**Статус: ПАРАЛЛЕЛЬНО С КДЛ (запускается самостоятельно)**

### Что
Лендинг/приложение: пользователь загружает ОАК (фото/PDF/ручной ввод),
получает интерпретацию + персональные рекомендации + "стоит проверить X".

### Кому
- Люди после профосмотра/чекапа ("что значат мои анализы?")
- ЗОЖ-аудитория
- Люди с хронической усталостью/симптомами

### Ценность
- Понятная расшифровка (не "гемоглобин 125 г/л", а "что это значит для вас")
- Конкретный следующий шаг (1 рекомендация, не 10)
- Быстро и бесплатно (базовый уровень)

### Фичи B2C (что отличает от GPT-ботов)

**MVP (неделя 3-4):**

| Фича | Что видит пользователь | Почему только у нас |
|---|---|---|
| Gauge вероятности | "Риск дефицита железа: 73%" | CatBoost даёт P, LLM — нет |
| SHAP-объяснение | "Почему: MCV ↓ + RDW ↑ + пол Ж" | Из модели, GPT-бот не может |
| Популяционное сравнение | "82% женщин 30-39 лет имеют MCV выше" | Данные NHANES, ни у кого нет |
| Светофор с контекстом | "MCV 79: не просто 'низкий', а паттерн дефицита" | Комбинации, не отдельные значения |
| "Один шаг" | "Сдайте ферритин (425 руб)" + кнопка | Конверсия, не перегруз |
| 1-2 совета | "Витамин C улучшает всасывание железа" | Контекстные, по результату |
| Дисклеймер | "Это не диагноз" | Обязательно |

**Версия 2 (месяц 2):**
- Загрузка PDF/фото ОАК (OCR)
- Личный кабинет + история
- Трекинг P(риска) по нескольким анализам
- Напоминание о пересдаче через 3 мес

**Версия 3 (месяц 3+):**
- Преддиабет (глюкоза + ИМТ)
- Платная подписка
- Telegram-бот

### Ключевое отличие от BloodGPT / Kantesti / GPT-ботов

```
Они: LLM сравнивает каждое значение с референсом ПО ОТДЕЛЬНОСТИ
     MCV=79 → "чуть ниже нормы". HGB=122 → "нижняя граница нормы".
     Итог: "в целом всё ок, контроль через 3-6 мес"
     → ПРОПУСК ДЕФИЦИТА

Мы:  CatBoost видит ПАТТЕРН из 14 параметров ОДНОВРЕМЕННО
     MCV+MCH+RDW+HGB+PLT+пол+возраст → P(дефицит) = 0.73
     → РЕКОМЕНДАЦИЯ: ферритин
     → PPV 65-70%: подтверждается у большинства
```

### БМ
- Freemium: gauge + светофор + 1 рекомендация бесплатно
- Платно (299 руб/мес): SHAP, перцентили, советы, динамика, история
- Партнёрка: CPA 50-100 руб за направление в КДЛ-партнёра

### Зачем это нужно стартапу
- Трафик и данные (валидация спроса)
- Proof of concept для КДЛ и инвесторов
- Независимость от партнёров (свой канал)
- Канал обратной связи: какие анализы загружают, что конвертит

### Данные на входе
Только лаб + пол + возраст (что человек сам введёт).
Модель: CatBoost с NaN для физ.данных (graceful degradation).
