```

КДЛ — не основной покупатель, а канал доступа.

Модель "КДЛ = просто канал" не мотивирует КДЛ участвовать.
v3 "Умный добор" даёт КДЛ прямую выгоду (+доп. выручка),
что делает партнёрство win-win.
```

### Рефлекс-тесты как SKU (unit economics, сохранить)

```
Пример: TSH Smart Reflex

ТТГ:     цена 1000р, себестоимость 200р, маржа 800р
Т4 св:   цена 1000р, себестоимость 200р, маржа 800р
Комплекс: цена 2000р, себестоимость 400р, маржа 1600р

Smart Reflex SKU: цена 1500р

Сценарий "здоровый" (ТТГ норм, стоп):
  Доход: 1500, расход: 250 (реагент+сервис), прибыль: 1250р (+450 vs простой ТТГ)

Сценарий "больной" (ТТГ отклонён -> рефлекс Т4):
  Доход: 1500, расход: 450, прибыль: 1050р (+250 vs простой ТТГ)

Главный риск: каннибализация
Если Smart-SKU начнут покупать те, кто раньше брал полную панель за 2000р:
- Потеря 550р на каждом таком пациенте

Защита от каннибализации:
- Контур доступа: Smart-SKU только в корпоративных/скрининговых каналах
- Сегментация: "есть жалобы/диагноз?" → полный профиль; "провериться?" → Smart
- Работодатели изначально не покупали полные панели → каннибализации нет

КОГДА ПРИГОДИТСЯ: при масштабировании на B2B / корпоративку.
Пока фокус на ферритине как первом добор-SKU.
```

## Идеи, которые всплывали, но не вошли в план

### "Корпоративный Паспорт Здоровья" — детальный user journey

```
1. Иван (сварщик) проходит обязательный профосмотр. За это платит Завод.
2. Клиника ставит штамп "Годен".
3. AI анализирует расширенный ОАК Ивана:
   - Hb норма, но MCV низкий, RDW высокий -> скрытый дефицит
   - Глюкоза верхняя граница + возраст + вес -> риск преддиабета
4. Ивану приходит СМС:
   "Иван, результаты вашего профосмотра проанализированы.
    Основные показатели в порядке.
    Обратите внимание: обнаружены изменения в показателях
    эритроцитов, которые иногда связаны с дефицитом железа.
    Рекомендуем: ферритин — 300 руб.
    [Записаться на добор — 1 клик]"

Формулировки (регуляторно безопасные):
- НЕ: "у вас риск анемии"
- ДА: "есть изменения в показателях эритроцитов"
- Обязательно: "это не диагноз", "рекомендуем обсудить с врачом"

КОГДА ПРИГОДИТСЯ: этап B2B, работодатели.
По сути — тот же "умный добор", но канал через работодателя, не через КДЛ.
```


## Принципы (из оригинального плана, по-прежнему актуальны)

1. **Не ставим диагнозы** -> "риск-сигналы", "рекомендации", дисклеймеры
2. **Не заменяем тесты** -> оцениваем вероятность полезности
3. **Не трогаем обязательный минимум 29н** -> работаем ПОСЛЕ него
4. **Хвост (доборы) оплачивается не нами** -> отдельный плательщик
5. **Не каннибализируем панели** -> контуры доступа, сегментация
6. **Режимы при дрейфе** -> модель умеет себя выключить
