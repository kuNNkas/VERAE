# IDA CASES: Сценарии дефицита железа — что предсказываем, как маршрутизируем

> Цены: Инвитро, 2025. Взятие крови: 310р за каждый визит.

---

## Что мы предсказываем

### Таргет: Body Iron (формула Кука)

```
Body Iron = -(log10(sTfR / ferritin) - 2.8229) / 0.1207

Если Body Iron < 0 → дефицит железа (y = 1)
Если Body Iron >= 0 → нет дефицита (y = 0)
```

**Почему Body Iron, а не просто ферритин?**

Ферритин — белок острой фазы. При воспалении (инфекция, аутоиммунное,
хроническое заболевание) ферритин растёт НЕЗАВИСИМО от запасов железа.

Пример:
```
Пациент А: ферритин = 45 нг/мл, СРБ = 2 мг/л
  → Реальные запасы ≈ ферритин. Норма.

Пациент Б: ферритин = 45 нг/мл, СРБ = 35 мг/л
  → Реальные запасы НИЖЕ, чем показывает ферритин.
    sTfR у него повышен → Body Iron < 0 → дефицит.
    Ферритин ВРЁТ.
```

sTfR (растворимый рецептор трансферрина) НЕ реагирует на воспаление.
Формула Кука (sTfR / ferritin) компенсирует ложь ферритина.

## Что берём на вход (фичи модели)

```
ОАК (уже сдан, 845р):
  HGB, HCT, MCV, MCH, MCHC, RDW, RBC
  PLT, MPV
  WBC, нейтрофилы%, лимфоциты%, моноциты%, эозинофилы%

Биохимия (если есть):
  Глюкоза, холестерин

Демография:
  Пол, возраст

Физ. данные (если есть, иначе NaN):
  ИМТ, систолическое АД
```

**Что НЕ берём на вход:**

| Параметр | Почему не на входе |
|---|---|
| Ферритин | Это то, что мы рекомендуем СДАТЬ. Если бы он уже был — модель не нужна |
| sTfR | Дорогой (2 445р), не рутинный. Используем только в формуле таргета при обучении |
| CRP | Не рутинный. Рекомендуем как добор при признаках воспаления в ОАК |
| Железо сыворотки | Вариабельно (±30% за сутки), малоинформативно для скрининга |
| ОЖСС / трансферрин | Не рутинные, дорогие |

## Что модель даёт на выходе

```
ВХОД (бесплатно — уже сдано):
  ОАК + пол + возраст
       ↓
  CatBoost (обученный на NHANES, таргет = Body Iron)
       ↓
ВЫХОД:
  1. P(дефицит железа) — вероятность от 0 до 1
  2. SHAP — какие параметры повлияли и насколько
  3. Перцентили — где пациент среди похожих (NHANES, по полу/возрасту)
  4. Флаг воспаления — WBC/нейтрофилы/PLT повышены?
  5. Рекомендация — что сдать и зачем (маршрутизация)
```

## Как модель учитывает воспаление БЕЗ CRP

Ключевой вопрос: CRP не на входе, как модель "знает" про воспаление?

**Ответ: модель ОБУЧЕНА на правильных метках + видит косвенные маркеры.**

### При обучении (NHANES):

```
Пациент из NHANES #4821:
  ОАК: MCV=76, RDW=17, WBC=13.5, нейтр=78%
  Ферритин: 52 нг/мл (формально "норма")
  sTfR: 4.8 мг/л (повышен)
  Body Iron = -1.3 → ДЕФИЦИТ (y = 1)

CatBoost видит: MCV↓ + RDW↑ + WBC↑ → y = 1
  Он УЧИТСЯ: "микроцитарный паттерн + воспаление = дефицит,
  НЕСМОТРЯ на то, что ферритин был бы нормальным"
```

Если бы мы тренировали на `ферритин < 30`, этот пациент получил бы метку y = 0 (ферритин 52, "норма"). Модель бы ПРОПУСТИЛА его.

Body Iron = -1.3 → правильная метка y = 1. Модель учится правильно.

### При предсказании (реальный пациент):

```
Новый пациент:
  ОАК: MCV=78, RDW=16.5, WBC=12.8, нейтр=75%

Модель: "Я видел похожих пациентов в NHANES.
  У них Body Iron был < 0 в 65% случаев."
  → P(дефицит) = 0.65

Дополнительно:
  WBC=12.8 > 10 и нейтр=75% > 70%
  → Флаг воспаления: ДА
  → Маршрут: ферритин + CRP (а не просто ферритин)
```

### Почему CBC достаточно для флага воспаления

| Маркер в ОАК | Что говорит о воспалении | Чувствительность |
|---|---|---|
| WBC > 10 × 10⁹/л | Лейкоцитоз — классический признак | Высокая |
| Нейтрофилы > 70% | Нейтрофилия — бактериальное / системное | Высокая |
| PLT > 400 × 10⁹/л | Реактивный тромбоцитоз | Средняя |
| Лимфоциты < 20% | Относительная лимфопения | Средняя |

Это НЕ замена CRP. Но для **решения "рекомендовать ли CRP вместе с ферритином"** — достаточно.

### Почему это САМЫЙ ценный сценарий

```
БЕЗ НАС:
  Врач смотрит ОАК: "гемоглобин 125, ну чуть ниже, бывает"
  Ферритин не назначает (не на слуху у терапевта при 29н)
  Даже если назначит ферритин: результат 45 нг/мл — "норма"
  А на самом деле Body Iron < 0 → дефицит замаскирован воспалением
  
  Пациент: устаёт, снижена работоспособность
  Никто не догадывается. Годами.

С НАМИ:
  Модель видит MCV↓ + RDW↑ + WBC↑ → P = 0.65
  Рекомендуем ферритин + СРБ → пациент знает контекст
  Врач получает два результата + пометку "возможна маскировка"
  → Правильное клиническое решение
```

### Почему СРБ, а не сразу sTfR?

| Параметр | Цена | Что даёт | Когда нужен |
|---|---|---|---|
| СРБ | 725р | Ответ: "можно ли верить ферритину?" | Всегда при флаге воспаления |
| sTfR | 2 445р | Прямой ответ: "есть ли дефицит?" | Только если СРБ↑ и ферритин 30-100 |

СРБ — дешёвая "страховка" (725р). В 60-70% случаев WBC↑ в ОАК не означает клинически
значимого воспаления (физ. нагрузка, стресс, курение). СРБ это уточняет.

Если СРБ нормальный → ферритин честный → вопрос закрыт за 1 945р.
Если СРБ повышен → пациент уходит к врачу с **двумя** результатами и контекстом.
Врач уже решает про sTfR — это не наша зона ответственности.

### Почему модель НЕ рекомендует ферритин

Это ключевой момент. Воспаление есть, но **эритроцитарный паттерн дефицита отсутствует**:
- MCV = 87 (норма, не снижен)
- MCH = 29 (норма)
- RDW = 13.2% (норма, не повышен)
- HGB = 132 (норма)

Модель обучена на Body Iron. Она знает, что дефицит проявляется через
MCV↓ + MCH↓ + RDW↑ ± HGB↓. Если этого паттерна нет — дефицита нет,
**независимо от WBC**.

```
Модель НЕ делает:
  "WBC повышен → значит воспаление → значит ферритин может врать
   → значит надо проверить" ← ЭТО НЕПРАВИЛЬНАЯ ЛОГИКА

Модель делает:
  "Эритроцитарные индексы в норме → дефицита нет → неважно,
   есть воспаление или нет" ← ЭТО ПРАВИЛЬНАЯ ЛОГИКА
```

Воспаление влияет на **интерпретацию ферритина**, но НЕ на **решение проверять ли ферритин**.
Решение проверять зависит от эритроцитарного паттерна, а не от наличия воспаления.

## FAQ: честные ответы

### "А если модель ошибётся и дефицита нет?" (False Positive)

```
Пациент сдал ферритин (1 220р), ферритин = 65 нг/мл → норма.
Деньги "потрачены впустую"? Нет:
  - Пациент получил УВЕРЕННОСТЬ (а не тревогу)
  - 1 220р vs 8 820р (если бы пошёл к врачу)
  - Формулировка: "рекомендация обоснована паттерном ОАК"
  - Это НЕ бесполезный тест — это исключение диагноза

PPV 60-70% означает: из 10 рекомендаций 6-7 подтвердятся.
Это ВЫШЕ, чем у типичного врачебного скрининга.
```

### "А если пропустим дефицит?" (False Negative)

```
Модель сказала P = 0.15 → зелёная зона → ничего не рекомендуем.
Но у пациента на самом деле Body Iron < 0.

Это возможно, но:
  1. Мы ничего не ОТМЕНЯЛИ. Пациент может сам пойти к врачу.
  2. Sensitivity >= 0.90 → пропускаем не более 10% случаев.
  3. Без нас пропустили бы 100% (никто не рекомендует ферритин по ОАК).
  4. Мы снижаем пропуск с ~100% до ~10%. Это прогресс.
```

### "Почему не рекомендовать sTfR всем при воспалении?"

```
sTfR = 2 445р + 310р = 2 755р.
Для скрининговой рекомендации — слишком дорого.
Конверсия SMS при чеке 2 755р будет ~10-15% (а не 30-35%).

Ферритин + СРБ = 1 945р — компромисс:
  В 60-70% случаев СРБ нормальный → ферритин честный → вопрос закрыт.
  Если СРБ высокий → пациент идёт к врачу с контекстом.
  Врач решает про sTfR. Это ЕГО ответственность.
```

### "Зачем СРБ, если можно просто смотреть на ферритин < 30?"

```
Ферритин < 30 при воспалении = дефицит ТОЧНО (даже воспаление не помогло).
Ферритин 30-100 при воспалении = НЕИЗВЕСТНО.

Без СРБ мы не знаем, "честный" это ферритин 45 или "надутый".
С СРБ:
  СРБ норм + ферритин 45 → честный → дефицита нет.
  СРБ ↑ + ферритин 45 → может быть "надутый" → нужен sTfR или врач.

725р за ответ на вопрос "можно ли верить ферритину?" — дёшево.
```
