# РИСКИ, ТУПИКИ И РАЗВИЛКИ

## Тупик 1: "ML-рефлекс для назначения анализов" как самостоятельный продукт

### Почему не работает
- Рефлекс-правила (if TSH < 0.3 -> FT4) есть у каждой КДЛ
- Простые rules дёшевы и понятны
- ML ради ML не продаётся

### Где сломалось
- КДЛ не платят за "умнее калькулятор"
- Нет боли, которую чистый рефлекс решает лучше, чем rule-engine

### Вывод
Рефлекс = функция внутри policy engine, а НЕ отдельный продукт.

## Тупик 2: "Предсказание вместо измерения"

### Почему не работает
- Юридически токсично
- Клинически опасно
- Репутационно смертельно
- "Не делайте тест, мы и так знаем" = конец

### Вывод
Модель НЕ заменяет тест. Модель оценивает вероятность полезности теста.
"Вероятность, что FT4 даст клинически значимую информацию — высокая/низкая"
-> маршрут, а не диагноз.

## Тупик 3: "Одна кривая на всё" (стационарное распределение)

### Почему не работает
- Распределение здоровые/больные НЕ стационарно
- Пандемии, мобилизации, сезонные ОРВИ, смена контингента
- Модель на одной кривой ломается при любом сдвиге

### Конкретная проблема
Если модель "фикс-ценой покрывает хвост":
- Много здоровых -> сверхприбыль
- Много больных -> убыток
- Это лотерея, а бизнес так не строят

### Решение
- Хвост оплачивается отдельным плательщиком (не КДЛ из маржи)
- Режимы работы (норма / перегруз / дефицит реагентов)
- ML-детект дрейфа -> авто-переключение режимов

## Тупик 4: "Экономить на минимуме 29н"

### Почему не работает
- Приказ 29н задаёт обязательный минимум обследований
- Ниже минимума нельзя опускаться
- ML не может "убрать ТТГ" если он входит в обязательный перечень
- Экономия на baseline = регуляторно мёртво

### Вывод
Пространство для модели — ТОЛЬКО:
1. Всё, что выше минимума (надстройка)
2. Частота и тайминг (когда повторять)
3. Интерпретация и маршрутизация ПОСЛЕ минимума

## Тупик 5: "КДЛ сами это сделают"

### Почему могут
- У Инвитро, Хеликса, МЕДСИ есть деньги, IT-отделы, данные

### Почему НЕ сделают
- Legacy-системы (ЛИС как "мамонты")
- Фокус на логистике, не на R&D
- **Конфликт интересов**: КДЛ выгодно делать больше тестов

### Решение
Не конкурировать с IT-отделом КДЛ.
Встать МЕЖДУ интересами: КДЛ хочет больше, страховая меньше, работодатель — оптимум.

## Тупик 6: CRP как рекомендованный добор

### Исходная логика
Модель предсказывает дефицит -> рекомендует "ферритин + CRP"
-> CRP повышен -> ферритин неинтерпретируем -> деньги потрачены зря

### Почему это провал
- CRP не в рутине, не в 29н
- Если CRP повышен, ферритин бесполезен
- "Зачем ваша модель, если добор не подтвердил ЖДА?"

### Решение (ключевой архитектурный инсайт)
Модель НЕ рекомендует "купить CRP". Модель ЗАМЕНЯЕТ потребность в CRP:
- Обучена на Body Iron (ферритин + sTfR) — уже скорректирована на воспаление
- CBC сам содержит сигналы воспаления (WBC, нейтрофилы, тромбоцитоз)
- "Предсказанный ферритин" точнее измеренного у пациентов с воспалением (MGH)

Маршрутизация:
- Нет воспаления в CBC -> добор: ферритин (достаточно)
- Есть воспаление в CBC -> добор: sTfR (а НЕ ферритин+CRP)
- Высокая вероятность + клиника -> эмпирическая терапия железом (0 руб на добор)

## Тупик 7: Каннибализация дорогих панелей

### Проблема
Smart-SKU (1500р) может каннибализировать полную панель (2000р):
- Пациент, который купил бы панель, берёт дешёвый smart-тест
- КДЛ теряет 550р на каждом

### Решения
1. **Контур доступа**: Smart-SKU только в корпоративных/скрининговых каналах
2. **Сегментация**: "есть жалобы/диагноз?" -> полный профиль; "провериться?" -> Smart
3. **Факт**: в корпоративке полные панели и так не покупались -> каннибализации нет

## Главные системные риски

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| КДЛ скопируют | Средняя | Высокое | Быстрый вход через МИС, не через КДЛ |
| Низкая конверсия B2C-докупок | Высокая | Среднее | Фокус на B2B (работодатель платит) |
| Регуляторные проблемы (SaMD) | Низкая (пока wellness) | Высокое | Дисклеймеры, не диагнозы |
| Дрейф популяции (пандемии) | Высокая | Высокое | Режимы, мониторинг, авто-откл |
| Интеграция в ЛИС/МИС тяжёлая | Высокая | Среднее | Начать с CSV/API, потом HL7/FHIR |
| "Это просто калькулятор" | Средняя | Высокое | Policy engine + экономическая оптимизация |

## Ключевые развилки (решённые)

| Вопрос | Ответ |
|---|---|
| ОМС или ДМС? | ОМС = данные. ДМС = деньги. |
| КДЛ или работодатель? | Работодатель платит, КДЛ выполняет |
| Предсказывать или рекомендовать? | Рекомендовать маршрут (не заменять тест) |
| Rule-based или ML? | ML для триажа/конверсии/дрейфа. Rules для красных флагов |
| Быстро или правильно? | White-label МИС быстро -> SaMD правильно потом |
