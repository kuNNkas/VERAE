# MVP PLAN: Реалистичный план запуска

## Стратегия входа: КДЛ/УКБ + B2C -> B2B работодатели -> МИС/страховые

### Почему так (а не через МИС)
- Есть реальные выходы на КДЛ и УКБ
- Нет выходов на МИС-вендоров
- B2C можно запустить самим (лендинг)
- Одна модель, три обёртки (graceful degradation через NaN)

---

## Этап 1: MVP (2-3 месяца)

### Цель
Доказать, что движок реально даёт пользу в КДЛ/УКБ + запустить B2C-канал.

### Продукт
Модуль "умной расшифровки и рекомендаций" для:
- ОАК (14 параметров)
- Ферритина (интерпретация с учётом воспаления)
- Глюкозы/ИМТ (преддиабет)
- Pre-analytical подсказки

### Сценарии MVP (2-3)

**Сценарий 1: ОАК -> риск ЖДА -> ферритин +/- CRP**
```
Вход: HGB, MCV, MCH, RDW, PLT, WBC, нейтрофилы + пол + возраст
Модель: CatBoost/XGBoost, обучена на NHANES (Body Iron как таргет)
Выход: P(iron deficiency) + флаг воспаления
Маршрут: "добор не нужен" / "ферритин" / "sTfR" / "эмпирическая терапия"
```

**Сценарий 2: Глюкоза/ИМТ -> риск преддиабета -> HbA1c**
```
Вход: глюкоза натощак, ИМТ, возраст, пол
Выход: P(prediabetes)
Маршрут: "норма" / "HbA1c рекомендован" / "срочная консультация"
```

**Сценарий 3: Pre-analytical подсказки**
```
Проверка: ферритин при воспалении, бессмысленные комбинации
Выход: предупреждение врачу/лаборанту
```

### Как строить модель

```
1. Данные: NHANES 2021-2023
   - CBC_L + BIOPRO_L + DEMO_L + FERTIN_L + TFR_L + BMX_L + BPX_L
   - Мерж по SEQN

2. Таргет: Body Iron < 0 мг/кг
   BI = -(log10(sTfR / ferritin) - 2.8229) / 0.1207

3. Фичи (Модель A "29н"):
   CBC: HGB, HCT, MCV, MCH, MCHC, RDW, RBC, PLT, MPV, WBC,
        нейтр%, лимф%, моноц%, эозин%
   BIOPRO: глюкоза, холестерин
   DEMO: пол, возраст
   BMX: ИМТ
   BPX: сист. АД

4. Алгоритм: CatBoost (лучший по литературе для этой задачи)
   - Stratified split по полу
   - Platt scaling для калибровки
   - SHAP для интерпретируемости

5. Метрики:
   - AUC-ROC >= 0.85
   - Sensitivity >= 0.90 при специфичности >= 0.60
   - Brier score, calibration curve
```

### Где взять первого партнёра
- КДЛ / УКБ (ЕСТЬ реальные выходы) -> pre-analytical валидатор
- Параллельно: B2C лендинг "загрузи ОАК" -> proof of demand + данные
- Потом: через кейс выходить на МИС-вендоров и работодателей

### Метрики пилота
- Конверсия: % людей, кто согласился на добор
- PPV: % доборов, которые подтвердили проблему
- Время врача: сокращение на интерпретацию
- Пересдачи: снижение на pre-analytical этапе

## Этап 2: Масштабирование КДЛ + рост B2C (3-6 мес после MVP)

### Схема
- КДЛ: расширение на 2-3 лаборатории, performance-fee за снижение пересдач
- B2C: платный уровень (паспорт здоровья, динамика, развёрнутые рекомендации)
- Партнёрка: реферальные ссылки на КДЛ-партнёра для доборов (CPA)

### KPI
- Количество КДЛ-партнёров
- Снижение пересдач (%) в пилотных КДЛ
- B2C: MAU, конверсия free->paid, конверсия в добор

## Этап 3: Корпоративный "биохим-аудит продуктивности" (6-12 мес)

### Продукт
"Модуль энергоэффективности сотрудников" через ДМС/профосмотры

### SKU

**SKU A (массовый): "Железо-энергия"**
- Вход: ОАК (из 29н) + анкета
- Add-on: ферритин +/- CRP -> только группе риска
- Плательщик: сотрудник или работодатель
- Цена: 300 руб add-on (при экономии 15 000 руб/мес продуктивности)

**SKU B (массовый): "Метаболика"**
- Вход: глюкоза + холестерин + ИМТ/талия + возраст
- Add-on: HbA1c / расширенные липиды
- Плательщик: сотрудник/работодатель

**SKU C (точечный, well-being): "Щитовидка Smart"**
- Вход: анкета + риски + триаж-модель
- Варианты:
  - TSH-first + доплата за FT4 при отклонении
  - Фикс-SKU с лимитом хвоста в B2B-контракте
- Плательщик: работодатель или сотрудник

### Каналы продаж
1. Через МИС-вендора (уже есть)
2. Через корпоративного провайдера ДМС (Лучи и подобные)
3. Через крупную частную сеть (Будь Здоров, Поликлиника.ру)
4. Напрямую к работодателю как wellbeing-надстройка

### Скрипт для HR-директора
```
"Вы тратите бюджет на 29н и ДМС. Но сотрудники всё равно уходят на больничные
или сидят вялые. Причина часто скрыта в анализах, которые 29н игнорирует.

Мы предлагаем Smart-надстройку:
1. Анализируем результаты 29н (бесплатно для вас)
2. Выявляем "группу риска" по продуктивности
3. Дообследуем точечно (Reflex-методом)

Результат: не лечите здоровых (экономия), но возвращаете ресурс больным (прибыль)."
```

## Этап 4: Страховые (12+ мес)

### Условие входа
- Есть внедрения через МИС
- Есть логи и данные пилотов
- Есть кейсы: "снизили пересдачи на 18%, сократили жалобы на 22%"

### Продукт
- B2B к страховой: контроль обоснованности назначений
- % от предотвращённых выплат
- Управление частотой мониторинга у групп риска

---

## Ключевые принципы (не нарушать)

1. **Не ставим диагнозы** -> "риск-сигналы", "рекомендации", дисклеймеры
2. **Не заменяем тесты** -> оцениваем вероятность полезности
3. **Не трогаем обязательный минимум 29н** -> работаем ПОСЛЕ него
4. **Хвост (доборы) оплачивается не нами** -> отдельный плательщик
5. **Не каннибализируем панели** -> контуры доступа, сегментация
6. **Режимы при дрейфе** -> модель умеет себя выключить
